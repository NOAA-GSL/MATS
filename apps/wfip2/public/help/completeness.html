Completeness

Completeness is a way of making your dataset meet a certain completeness criteria, expressed as a percentage.

The data for this tool is gathered in groups. For example, all of the wind speed data is grouped by level, bounded by a top and a bottom level for every individual site, and restricted to a certain time range. The value at a given point in time is statistically derived, defaulting to the mean. For example, if the statistic is "none" and the variable is "wind speed" then the values on the graph will be the mean of the wind speeds for every level within the top and bottom boundary for each time.

levels

It is possible that, at a certain time, one site might have only reported data for a subset of the levels which it is normally capable of. This is where completeness comes in handy. If you consider the maximum set of levels possible for the data set that you have selected as a 'basis set' of levels then you can choose a percentage of completeness that must be met for each data point, derived by comparing each point against that basis, in order to have the data included into the mean.

For example: there may be 10 distinct levels between a bottom of 40 and a top of 200. If I ask for 20 sites, and one of those sites, at a given time, only has 5 levels reported, then that point is 50% complete. Had I specified a completeness of 60% the data routine would throw away that set of levels and not consider it into the mean calculation for that point. If I set a completeness of 100 then each site at every time must have the maximum possible number of levels reported. If I specify a completeness of 0 then all levels reported will be considered for each site into the mean calculation.

For the data set below, a completeness of 100 would result in rows 3, 4, and 5 being discarded, while a completeness of 80 would result in the data for row 3 being discarded and the remainder retained for use in the calculation.

sites

Sometimes more than one site can be used to derive the data. Similarly to levels, if the total possible number of sites for the data set is 10 and a site completeness of 80 is specified, each data point that does not have at least 8 sites (80 percent of the total possible) reporting data will be discarded. A completeness of 0 would cause the data routine to use all of the sites.
