The level and site matchers are used to match the data plotted on each axis agianst the other based on a percentage set by the user

When the level or site matcher is selected the data on both axes will be cut down to match the other as closely as desired. Due to inconsistencies collecting data from models and observations some data points may be skewed or misleading. However, only plotting data points that match exactly with every other point often leaves little or no data left to be plotted. The goal of the axis matching quality selector is to give the user control over the trade-off between the amount and the consistency of the data.

For Example

If you are plotting wind speed on the x-axis and the y-axis for three different hours on a given day your data will look something like this:

x-axis

8:00 -   wind_speed [5.3, 7.6, 9.3, 15.4, 19.3]                        at height [0, 20, 40, 60, 80]

12:00 - wind_speed [3.2, 4.7, 6.8, 10.5, 14.2, 18.8, 21.3]       at height [0, 20, 40, 60, 80, 100, 120]

4:00 -   wind_speed [2.5, 6.4, 7.5, 9.9, 13.8]                          at height [0, 20, 40, 60, 80]

y-axis

8:00 -   wind_speed [3.4, 6.7, 8.1, 13.2]                                 at height [0, 20, 40, 60]

12:00 - wind_speed [2.2, 4.2, 7.9, 11.5, 15.8, 20.5]                at height [0, 20, 40, 60, 80, 100]

4:00 -   wind_speed [2.5, 6.4]                                                at height [0, 20]

 

Now you want to match the axes by level (height). When you plot a curve with the level axis matcher selected, three things happen:

1. All the heights of both axis are unioned, in this case x-axis: [0, 20, 40, 60, 80, 100, 120] and             y-axis: [0, 20, 40, 60, 80, 100]

2. The intersection of the two is taken, in this example [0, 20, 40, 60, 80, 100]. This is now your             common basis which all the data will be compared against.

3. The common basis will be compared against every data point on both axes and the                           percentage of values contained in both the point and the common basis will be calculated. The         first point on the x-axis contains 5 out of 6 of the values in the common basis so the percentage       returned will be 5/6 or 83%. The second point on the x-axis contains 6 out of 6 values and will           return 100%. The last point on the y-axis contains 2 out 6 values and will return 33%.

    If the percentage returned is lower than the percentage set in the axis matching quality then the         point will be discarded and will not show up on the plot. If the axis matching quality is set to 0% all     points will be plotted as if no matcher were selected, if set to 100% it is likely that little-to-no data     will be plotted.
